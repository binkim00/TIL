## 🧪 테스트 4-1 — 채점 및 보정

### 🔹 코드

```python
numbers = [1,2,3]

defadd_number(lst):
	lst.append(4)

add_number(numbers)

```

### ❓ 질문 1

이 코드 실행 이후 리스트 `numbers`의 최종 값은 무엇인가?

### ✅ 답변

> [1, 2, 3, 4]
> 

✔ **정답**

👉 `append(4)`가 실행되어 리스트에 값이 추가된다.

---

### ❓ 질문 2

`add_number` 함수 안에서 사용한 `lst`는

`numbers`와 같은 객체인가, 다른 객체인가?

### ✅ 답변

> 다른 객체
> 

❌ **오답**

### 📌 정확한 정리

👉 `lst`와 `numbers`는 **같은 리스트 객체**를 가리킨다.

함수 인자로 리스트를 전달하면 **참조가 전달**된다.

---

### ❓ 질문 3

이 함수 호출 이후에도 `numbers`가 변경된 이유를

“함수 + 리스트” 관점에서 한 문장으로 설명하시오.

### ✅ 답변

> add_number 함수를 호출해서 list에 4를 추가했음
> 

🟡 **의도는 맞음, 표현 보정 필요**

### 📌 정확한 정리

👉 함수에 전달된 리스트는 **같은 객체**이므로,

함수 내부에서 `append`로 변경하면 원본 리스트도 함께 변경된다.

## 🔍 테스트 4-1 핵심 정리

```markdown
- 리스트를 함수 인자로 넘기면 객체의 참조가 전달된다
- append, remove 등은 원본 리스트를 직접 변경한다
- lst = [...] 는 새로운 리스트를 만드는 것이지 원본을 바꾸지 않는다
- "객체를 수정했는가"와 "변수를 재할당했는가"를 구분해야 한다

```

