## 🧪 테스트 2-확장 — 전역 변수 vs 지역 변수

```python
count =0

defincrease():
    count =10
    count +=1
print(count)

increase()
print(count)

```

---

### ❓ 질문 1

`increase()` 함수 안에서 출력되는 `count` 값은 얼마인가?

### ✅ 답변

> 11이다.
> 

✔ **정답**

👉 함수 안에서 `count = 10`으로 지역 변수를 만들고, `count += 1`로 11이 된 뒤 출력한다.

---

### ❓ 질문 2

함수 실행이 끝난 뒤 `print(count)`에서 출력되는 `count` 값은 얼마인가?

### ✅ 답변

> 0이다.
> 

✔ **정답**

👉 함수 밖의 `count`는 전역 변수이며, 함수 안에서 만든 지역 변수 `count`는 전역 변수에 영향을 주지 않는다.

---

### ❓ 질문 3

함수 안의 `count = 10`은 함수 밖의 `count`와 같은 변수인가, 다른 변수인가?

### ✅ 답변

> 다른 변수이다.
> 

✔ **정답**

👉 함수 안의 `count`는 지역 변수로 새로 생성된 변수이다.

---

### ❓ 질문 4

`global count`가 없는데도 에러가 나지 않는 이유는 무엇인가?

### ✅ 답변

> 함수 안에서 count = 10으로 지역 변수를 먼저 만들었기 때문이다.
> 

✔ **정답**

👉 지역 변수를 먼저 생성했기 때문에 `count += 1`에서 읽기/더하기가 가능하다.

---

### ❓ 질문 5

함수 안을 아래처럼 바꾸면 어떤 일이 발생하는가?

```python
defincrease():
    count +=1
print(count)

```

### ✅ 답변

> 에러가 발생한다.
> 

✔ **정답**

👉 `count += 1`은 내부적으로 `count = count + 1`로 동작하며, 오른쪽 `count`를 먼저 읽어야 한다.

👉 그러나 `global` 선언도 없고, 지역 변수로 미리 만든 적도 없어서 “아직 생성되지 않은 지역 변수를 먼저 읽는다”가 되어 에러가 발생한다.

## 📌 핵심 요약 (md용)

```markdown
- 함수 안에서 대입(count = 10)을 하면 지역 변수가 생성된다
- 지역 변수는 함수가 끝나면 사라지고, 전역 변수에 영향을 주지 않는다
- global이 없을 때 count += 1만 쓰면, 지역 변수를 만들기 전에 읽게 되어 에러가 난다

```
