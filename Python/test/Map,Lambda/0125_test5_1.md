## 🧪 테스트 5-2 — map + lambda 읽기

### 🔹 코드 A (기존 함수)

```python
numbers = [1,2,3,4]

defdouble(n):
return n *2

result =list(map(double, numbers))
print(result)

```

---

### 🔹 코드 B (lambda 사용)

```python
numbers = [1,2,3,4]

result =list(map(lambda n: n *2, numbers))
print(result)

```

## 🔍 오늘 map + lambda에서 가장 중요한 핵심

영빈님이 오늘 헷갈렸던 모든 지점은

**이 한 줄로 정리됩니다.**

> map은 “원소 하나 → 결과 하나” 함수가 있어야 동작한다.
lambda는 그 함수를 짧게 쓰는 방법일 뿐이다.
>

## map / lambda 핵심 정리

### 1. map의 역할
map은 리스트의 각 요소를 하나씩 함수에 전달하여,
그 반환값들로 새로운 결과를 만든다.

- 원본 리스트를 수정하지 않는다
- 리스트 길이만큼 함수가 자동으로 호출된다

### 2. map에 들어가는 함수의 조건
map에 전달되는 함수는 반드시
- 입력 하나를 받고
- 결과 하나를 return 해야 한다

즉,
n 하나 → 결과 하나
형태의 함수만 사용할 수 있다.

### 3. lambda의 의미
lambda는 이름이 없는 한 줄짜리 함수 표현식이다.
기존에 def로 만들던 간단한 함수를
짧게 표현하기 위한 문법이다.

### 4. def와 lambda의 관계
아래 두 코드는 동일한 의미를 가진다.

def double(n):
    return n * 2

lambda n: n * 2

차이는 길이와 이름의 유무뿐이다.

### 5. 언제 lambda를 쓰는가
- 함수가 한 줄로 끝나고
- 한 번만 사용되며
- map, filter 같은 곳에 바로 전달할 때
lambda를 사용한다.
