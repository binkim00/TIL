## 🧪 테스트 5-1 — map의 역할 이해 (for문 대응)

### 🔹 코드 A (기존 for문)

```python
numbers = [1,2,3,4]

result = []
for nin numbers:
	result.append(n *2)

print(result)

```

---

### 🔹 코드 B (map 사용)

```python
numbers = [1,2,3,4]

def double(n):
return n *2

result =list(map(double, numbers))
print(result)

```

### ❓ 질문 1

코드 A와 코드 B의 최종 출력 결과는 같은가, 다른가?

### ✅ 답변

> 같음
> 

✔ **정답**

```python
[2,4,6,8]

```

👉 `for문`이든 `map`이든 **의미는 동일**하다.

## 🔑 질문 5 핵심 설명 (중요)

`map`에 들어가는 함수는 **반드시** 이 조건을 만족해야 합니다.

### ✅ 핵심 조건 (이 한 줄)

> 입력 하나를 받아서
결과 하나를 return 하는 함수
> 

즉:

```
n 하나 → 결과 하나

```

그래서 이런 함수만 map에 쓸 수 있습니다:

```python
def double(n):
return n *2

```

❌ 이런 함수는 map에 부적합합니다:

```python
def wrong(lst):
	lst.append(4)# 입력이 리스트 전체
# return 없음

```

---

## 🔍 지금 헷갈린 포인트 정확히 짚기

영빈님이 **C를 고른 이유**는 이 생각 때문입니다:

> “map도 결과가 바뀌니까 수정하는 거 아닌가?”
> 

하지만 차이는 이겁니다:

- ❌ 수정: 원본 리스트를 바꿈
- ✅ map: **원본 그대로 두고 새 결과를 만듦**

```python
numbers = [1,2,3,4]
result =list(map(double, numbers))

print(numbers)# [1, 2, 3, 4]
print(result)# [2, 4, 6, 8]

```

👉 원본 안 바뀜 = **수정 아님**
