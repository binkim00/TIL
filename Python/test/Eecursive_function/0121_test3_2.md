## 🧪 테스트 3-2 — 반환값은 있지만 계산은 없는 재귀

```python
def func(n):
    if n ==0:
        return 0
return func(n -1)

result = func(3)

```

---

### ❓ 질문 1

이 함수는 무한 재귀일까요, 정상 종료될까요?

### ✅ 답변

> 정상 종료된다.
> 

✔ 정답

👉 `n`이 `3 → 2 → 1 → 0`으로 계속 줄어들기 때문에

종료 조건 `if n == 0:`에 도달하여 정상 종료된다.

---

### ❓ 질문 2

정상 종료된다면, 어디에서 멈추는가?

### ✅ 답변

> func(0)이 호출될 때 멈춘다.
> 

✔ 정답

👉 `func(0)`에서 조건이 참이 되어 `return 0`이 실행되고,

그 이후로는 더 이상 재귀 호출이 발생하지 않는다.

---

### ❓ 질문 3

이 함수는 총 몇 번 호출되는가?

### ✅ 답변

> 총 4번 호출된다.
> 

✔ 정답

호출 흐름은 다음과 같다.

```
func(3) → func(2) → func(1) → func(0)

```

---

### ❓ 질문 4

`result`에는 어떤 값이 저장되는가?

### ✅ 답변

> 0이 저장된다.
> 

✔ 정답

👉 가장 마지막 호출인 `func(0)`에서 반환된 `0`이

그대로 위로 전달되어 최종 결과가 된다.

---

### ❓ 질문 5

이 함수에는 `return 값`이 있는데도

“계산을 하지 않는다”고 말할 수 있는 이유는 무엇인가?

### ✅ 답변

> 반복 호출만 있을 뿐, 값을 변경하는 코드가 없기 때문이다.
> 

✔ 정답

👉 이 함수는 매 호출마다 **같은 값(0)을 그대로 반환**할 뿐,

`n`을 더하거나 곱하는 등의 계산을 수행하지 않는다.

---

### ❓ 질문 6

이 함수와 테스트 3-1의 함수의 가장 큰 차이점은 무엇인가?

### ✅ 답변

> 결과를 할당하는 변수가 있다.
> 

✔ 결론은 맞다. 표현을 정리한다.

### 📌 정확한 정리

- 테스트 3-1은 반환값이 없는 재귀이다
- 테스트 3-2는 반환값은 있지만, 계산이 없는 재귀이다
- 테스트 3-2에서는 반환된 값이 `result` 변수에 할당된다

---

## 🔍 테스트 3-2 핵심 정리

```markdown
- 반환값이 있는 재귀라고 해서 반드시 계산을 하는 것은 아니다
- 재귀 호출 구조와 반환 구조는 서로 분리해서 생각해야 한다
- 값이 바뀌지 않으면 계산이 없는 재귀가 된다
- 최종 반환값은 종료 조건에서 결정된다

```
