-## 🧪 테스트 3 — 재귀 함수 (기본 흐름)

```python
def func(n):
    if n ==1:
        return 1
    return n + func(n -1)

result = func(3)

```

---

### ❓ 질문 1

이 코드에서 재귀 호출이 시작되는 지점은 어디인가?

### ✅ 답변

> func(3)이다.
> 

✔ 정답

👉 `result = func(3)`에서 최초로 함수 호출이 시작된다.

---

### ❓ 질문 2

`func(3)`이 실행될 때 실제로 호출되는 함수들을 호출 순서대로 나열하시오.

### ✅ 답변

> func(3) → func(2) → func(1)
> 

✔ 정답

👉 `func(1)` 이후에는 더 이상 재귀 호출이 발생하지 않는다.

---

### ❓ 질문 3

어느 시점에서 재귀 호출이 멈추는가? 그 이유는 무엇인가?

### ✅ 답변

> func(1)에서 멈춘다.
> 
> 
> `n == 1` 조건이 만족되기 때문이다.
> 

✔ 정답

👉 이 조건을 종료 조건(base case)이라 한다.

---

### ❓ 질문 4

`func(1)`은 어떤 값을 반환하는가?

### ✅ 답변

> 1을 반환한다.
> 

✔ 정답

👉 종료 조건에 해당하므로 즉시 반환한다.

---

### ❓ 질문 5

`func(2)`는 어떤 계산을 거쳐 어떤 값을 반환하는가?

### ✅ 답변

> func(2)는 2 + func(1)을 계산한다.
> 
> 
> `func(1)`이 1을 반환하므로, `2 + 1 = 3`을 반환한다.
> 

✔ 정답

---

### ❓ 질문 6

최종적으로 `result`에는 어떤 값이 저장되는가?

### ✅ 답변

> 6이 저장된다.
> 

✔ 정답

👉 계산 과정은 다음과 같다.

- `func(1) = 1`
- `func(2) = 2 + 1 = 3`
- `func(3) = 3 + 3 = 6`

---

### ❓ 질문 7

이 재귀 함수에서 절대 없어지면 안 되는 코드 한 줄은 무엇인가?

그 줄이 없으면 어떤 문제가 발생하는가?

### ✅ 답변

```python
if n ==1:
return 1

```

✔ 정답

👉 이 줄이 없으면 종료 조건이 사라져 무한 재귀가 발생한다.

---

### ❓ 질문 8

이 함수의 전체 동작을 “호출 단계 / 반환 단계”로 구분하여 설명하시오.

### ✅ 답변

> 호출 단계에서는 n이 1이 될 때까지 func(n - 1)이 반복 호출된다.
> 
> 
> 반환 단계에서는 `func(1)`에서 반환된 값을 이용해
> 
> 이전 호출 순서대로 `n`을 더하며 결과를 계산한다.
> 

✔ 정답

---

## 📌 테스트 3 핵심 요약 (md용)

```markdown
- 재귀는 함수가 자기 자신을 호출하는 구조이다
- 반드시 종료 조건(base case)이 필요하다
- 호출 단계에서는 인자가 점점 줄어든다
- 반환 단계에서 계산이 누적된다
- 종료 조건이 없으면 무한 재귀가 발생한다

```
