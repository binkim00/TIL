## 🧪 테스트 3-1 — 더하기 없는 재귀 (착시 제거)

```python
def func(n):
    if n ==0:
        return
    func(n -1)

func(3)

```

---

### ❓ 질문 1

이 함수는 무한 재귀일까요, 정상 종료될까요?

### ✅ 답변

> 정상 종료된다.
> 

✔ 정답

👉 `n`이 `3 → 2 → 1 → 0`으로 계속 줄어들기 때문에

종료 조건인 `if n == 0:`에 반드시 도달한다.

따라서 이 재귀 함수는 정상 종료된다.

---

### ❓ 질문 2

정상 종료된다면, 어디에서 멈추는가?

### ✅ 답변

> func(0)에서 멈춘다.
> 

✔ 정확한 정답

👉 `func(0)`에서 조건이 참이 되어 `return`이 실행되고,

그 이후로는 더 이상 재귀 호출이 발생하지 않는다.

---

### ❓ 질문 3

이 함수는 총 몇 번 호출되는가?

### ✅ 답변

> 총 4번 호출된다.
> 

✔ 정답

호출 흐름은 다음과 같다.

```
func(3) → func(2) → func(1) → func(0)

```

---

### ❓ 질문 4

이 재귀 함수에는 “계산되는 값”이 있는가?

### ✅ 답변

> 계산되는 값은 없다.
> 

✔ 결론은 정답

### 📌 정확한 개념 정리

- 이 함수에는 `return`이 있지만 **반환값이 없다**
- 따라서 결과를 누적하거나 계산하지 않는다
- 이 재귀는 값을 만드는 구조가 아니라
    
    **호출 후 종료되는 “동작 중심 구조”이다**
    

👉 “result 변수가 없다”는 표현도 감각적으로는 맞는 설명이다.

---

## 🔍 테스트 3-1 핵심 정리

```markdown
- 재귀는 반드시 값을 계산해야만 의미가 있는 것은 아니다
- 종료 조건과 함께 호출 구조만으로도 정상적인 재귀가 된다
- 값이 없는 재귀는 “동작만 수행하고 끝나는 구조”이다
- 재귀의 핵심은 호출 흐름과 종료 지점이다

```
