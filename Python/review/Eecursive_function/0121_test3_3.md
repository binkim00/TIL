## 🧪 테스트 3-3 — 재귀 → 반복문 변환

### 🔹 재귀 코드

```python
def func(n):
	if n ==0:
		return 0
	return func(n -1)

result = func(3)

```

---

### ❓ 질문 1

이 재귀 함수에서 종료 조건은 무엇인가?

### ✅ 답변

> func(0)이 호출되어 n == 0이 될 때이다.
> 

✔ 정답

👉 `if n == 0:` 조건이 만족되면 `return 0`이 실행되며 재귀가 종료된다.

---

### ❓ 질문 2

이 함수가 재귀 호출을 하면서 실제로 변하는 값은 무엇인가?

### ✅ 답변

> func(n)의 n 값이다.
> 

✔ 정답

👉 재귀 호출마다 `n - 1`이 전달되며, `n`이 점점 줄어든다.

---

### ❓ 질문 3

아래 중 이 재귀 함수와 동작이 같은 반복문 구조는 무엇인가?

### ✅ 답변

> A
> 

✔ 정답

```python
n =3
while n >0:
    n -=1
result =0

```

👉 `n`이 0이 될 때까지 감소하며 반복이 종료되는 구조가 재귀와 동일하다.

---

### ❓ 질문 4

해당 반복문을 선택한 이유는 무엇인가?

### ✅ 답변

> result 값이 0이 될 때까지 호출해서이다.
> 

🟡 의도는 맞다. 표현을 정리한다.

### 📌 정확한 정리

👉 이 재귀 함수는 값을 계산하지 않고

`n`이 0이 될 때까지 호출만 수행한다.

반복문 A 역시 `n`이 0이 될 때까지 감소시키며 종료된다.

---

### ❓ 질문 5

이 재귀 함수가 반복문으로 바뀌면서도 변하지 않는 핵심 개념은 무엇인가?

### ✅ 답변

> n의 값이 종료 기준이 될 때까지 진행된다는 점이다.
> 

✔ 정답

👉 재귀와 반복문 모두

- 종료 조건이 존재하고
- 그 조건을 향해 값이 점점 변한다는 구조는 동일하다.

---

## 🔍 테스트 3-3 핵심 정리

```markdown
- 재귀와 반복문은 동일한 종료 조건을 가질 수 있다
- 재귀 호출에서 변하던 값은 반복문에서도 그대로 유지된다
- 계산이 없는 재귀는 반복문에서도 값 변화 없이 표현 가능하다
- 핵심은 “어떤 값이 언제까지 변하는가”이다

```
