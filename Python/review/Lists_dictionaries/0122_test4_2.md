### ❓ 질문 1

코드 A 실행 결과로 출력되는 `numbers`와 `result`는 각각 무엇인가?

```python
numbers = [1,2,3]

def add_four(lst):
	lst.append(4)

result = add_four(numbers)
print(numbers)
print(result)

```

### ✅ 답변

> number = [1,2,3,4] / result = [1,2,3,4]
> 

❌ **오답 (result 부분)**

### 📌 정확한 정리

```python
print(numbers)# [1, 2, 3, 4]
print(result)# None

```

👉 `append`는 리스트를 **수정만 하고 반환값이 없다**.

👉 `return`이 없으므로 `result`에는 `None`이 들어간다.

### ❓ 질문 3

코드 B 실행 결과로 출력되는 `numbers`와 `result`는 각각 무엇인가?

```python
numbers = [1,2,3]

def add_four(lst):
	new_list = lst + [4]
return new_list

result = add_four(numbers)
print(numbers)
print(result)

```

### ✅ 답변

> number = [1,2,3,4] / result = [1,2,3,4]
> 

❌ **오답 (`numbers` 부분)**

### 📌 정확한 정리

```python
print(numbers)# [1, 2, 3]
print(result)# [1, 2, 3, 4]

```

👉 `lst + [4]`는 **새 리스트를 생성**한다.

👉 원본 `numbers`는 변경되지 않는다.

### ❓ 질문 5

코드 C에서 `user`와 `result`의 관계를 객체 관점에서 설명하시오.

```python
user = {"name":"kim","age":20}

def increase_age(info):
	info["age"] +=1
return info

result = increase_age(user)
print(user)
print(result)

```

### ✅ 답변

> user와 result는 안에 들어있는 값은 같아보이지만 result안에 있는 값은 새로 할당된 딕셔너리임
> 

❌ **오답**

### 📌 정확한 정리

```python
print(user)# {'name': 'kim', 'age': 21}
print(result)# {'name': 'kim', 'age': 21}

```

👉 `user`와 `result`는 **같은 딕셔너리 객체**이다.

```python
defincrease_age(info):
	info["age"] +=1
return info

```

- `info`는 `user`와 같은 객체
- 객체를 수정한 뒤 **그 객체를 그대로 반환**

❗ 새 딕셔너리를 만든 코드가 아니다.

## 🔍 테스트 4-3 핵심 정리

```markdown
- append는 리스트를 수정하지만 반환값은 None이다
- return이 없으면 함수 결과는 항상 None이다
- 리스트를 새로 만들어 반환하면 원본은 변경되지 않는다
- 객체를 수정하고 반환하면 원본과 반환값은 같은 객체다
- “수정 + return”과 “새 객체 return”은 전혀 다르다

```
