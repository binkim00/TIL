## 🧪 테스트 1 — 함수 + 인자 (핵심)

```python
defadd(a, b):
return a + b

result = add(3,5)

```

---

### ❓ 질문 1

`a`에는 무엇이 들어가는가?

### ✅ 답변

> a에는 3이 들어간다.
> 

✔ **정답**

👉 함수가 호출될 때 `add(3, 5)`에서

첫 번째 인자 `3`이 매개변수 `a`에 전달된다.

---

### ❓ 질문 2

`b`에는 무엇이 들어가는가?

### ✅ 답변

> b에는 5가 들어간다.
> 

✔ **정답**

👉 두 번째 인자 `5`가 매개변수 `b`에 전달된다.

---

### ❓ 질문 3

`return`은 어디로 가는가?

### ❌ 답변

> a + b의 값이 result 안으로 간다.
> 

⚠️ **의미는 맞지만, 표현이 정확하지 않다.**

---

### ✅ 정확한 개념

> return 값은 변수로 직접 가는 것이 아니라,
> 
> 
> **함수가 호출된 자리로 돌아간다.**
> 

즉,

```python
result = add(3,5)

```

이 줄에서

- `add(3,5)`라는 **함수 호출 자리**가
- `return` 값인 `8`로 **치환**되고
- 그 결과 `8`이 `result`에 저장된다.

👉 **return은 “함수 호출 자리”로 돌아간다.**

👉 이 개념이 매우 중요하다.

---

### ❓ 질문 4

`result`는 언제 생성되는가?

### ✅ 답변

> add 함수가 호출되고, 결과가 나온 뒤 생성된다.
> 

✔ **의미는 맞다. 표현을 조금 더 정확히 정리한다.**

---

### ✅ 정확한 표현

> result는
> 
> 
> `add(3,5)` 함수의 실행이 **완전히 끝난 후**,
> 
> `return` 값(8)을 받아 **대입되면서 생성된다.**
> 

---

## 📌 테스트 1 핵심 요약 (md용)

```markdown
- 인자는 함수 호출 시 매개변수로 전달된다
- return 값은 변수로 가는 것이 아니라 함수 호출 자리로 돌아간다
- 함수 실행이 끝난 뒤 return 값이 대입되며 변수가 생성된다

```
